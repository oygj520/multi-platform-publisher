## 🚀 快速开始

### 1. 安装依赖

**首次使用需要安装依赖**：

```batch
cd E:\openclaw-projects\multi-platform-publisher
pip install -r requirements.txt
```

### 2. 准备浏览器

确保 Chromium 浏览器在以下路径：
```
E:\chrome-win\chrome.exe
```

如果还没有，可以：
- 手动下载 Chromium 到该位置
- 或使用 Playwright 自动下载：`playwright install chromium`

### 3. 启动应用

```batch
run.bat
```

或：
```batch
python app.py
```

### 4. 配置 Cookie

**首次使用必须配置 Cookie**，有两种方式：

#### 方式一：扫码登录（推荐 ⭐）

1. 进入「Cookie 管理」标签页
2. 选择要登录的平台
3. 点击 **「扫码登录」** 按钮
4. 使用手机 APP 扫描二维码
5. 在手机上确认登录
6. 等待「✅ 登录成功」提示

> 💡 **优势**：简单快捷，无需手动复制 Cookie，30 秒内完成

#### 方式二：手动配置 Cookie

1. 进入「Cookie 管理」标签页
2. 点击对应平台的「编辑」按钮
3. 粘贴 Cookie 数据
4. 点击「保存 Cookie」

> 📝 **提示**：详细步骤见下方「如何获取 Cookie」章节

### 5. 发布内容

1. **编写内容**
   - 输入标题
   - 使用 Markdown 编写正文
   - 选择封面图片（可选）

2. **选择平台**
   - 勾选要发布的平台

3. **一键发布**
   - 点击「🚀 一键发布」
   - 等待发布完成

### Cookie 管理

#### 📱 扫码登录（推荐 ⭐）

**什么是扫码登录？**

扫码登录是一种简单快捷的登录方式，无需手动复制 Cookie。只需使用手机 APP 扫描电脑屏幕上的二维码，即可自动完成登录并保存 Cookie。

**使用步骤**：

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 点击扫码登录 │ →  │ 手机扫描     │ →  │ 自动保存     │
│   按钮       │    │ 二维码       │    │ Cookie       │
└──────────────┘    └──────────────┘    └──────────────┘
```

**详细步骤**：

1. **进入 Cookie 管理**
   - 打开应用主界面
   - 点击顶部标签页 **「Cookie 管理」**

2. **选择平台**
   - 找到要登录的平台卡片（知乎/小红书/快手/抖音）
   - 确认平台状态显示为「未登录」

3. **点击扫码登录**
   - 点击对应平台的 **「扫码登录」** 按钮
   - 确认弹出的对话框，点击「确定」

4. **等待浏览器弹出**
   - 系统会自动启动 Chromium 浏览器
   - 浏览器窗口会显示平台的登录页面
   - 页面上会显示登录二维码

5. **手机扫描二维码**
   - 打开手机上的对应平台 APP
   - 找到「扫一扫」功能（位置见下方表格）
   - 对准电脑屏幕上的二维码扫描

6. **手机确认登录**
   - 手机上会显示登录确认提示
   - 点击「确认登录」或「允许」

7. **等待登录成功**
   - 浏览器会自动检测登录状态
   - 看到「✅ 登录成功」提示
   - Cookie 已自动保存并加密存储

**各平台扫码入口**：

| 平台 | 手机 APP 扫码入口 | 二维码位置 |
|------|-----------------|-----------|
| 知乎 | 知乎 APP → 首页右上角扫一扫图标 | 页面中央 |
| 小红书 | 小红书 APP → 首页左上角扫一扫图标 | 页面中央 |
| 快手 | 快手 APP → 首页左上角扫一扫图标 | 页面中央 |
| 抖音 | 抖音 APP → 首页右上角扫一扫图标 | 页面中央 |

#### 🔧 手动获取 Cookie（备选）

如果扫码登录不可用，可以手动获取 Cookie：

**知乎**:
1. 访问 https://www.zhihu.com
2. 登录账号
3. F12 打开开发者工具
4. Application → Cookies → https://www.zhihu.com
5. 右键 → Copy All

**小红书**:
1. 访问 https://www.xiaohongshu.com
2. 登录账号
3. F12 → Application → Cookies
4. 复制 Cookie 值

**快手**:
1. 访问 https://www.kuaishou.com
2. 登录账号
3. F12 → Application → Cookies
4. 复制 Cookie 值

**抖音**:
1. 访问 https://www.douyin.com
2. 登录账号
3. F12 → Application → Cookies
4. 复制 Cookie 值

#### Cookie 格式

支持两种格式：

1. **字符串格式**:
```
key1=value1; key2=value2; key3=value3
```

2. **JSON 格式**:
```json
[
  {"name": "key1", "value": "value1", "domain": ".zhihu.com"},
  {"name": "key2", "value": "value2", "domain": ".zhihu.com"}
]
```

## ⚠️ 注意事项

### 1. Cookie 安全

- ✅ Cookie 使用 Fernet 加密存储
- ✅ 存储位置：`~/.multi_platform_publisher/cookies.db`
- ✅ 加密密钥：`~/.multi_platform_publisher/.key`
- ❌ 不要分享 Cookie 给他人
- ❌ 不要在公共电脑上使用

### 2. 发布限制

| 平台 | 建议间隔 | 注意事项 |
|-----|---------|---------|

- 需要稳定的网络连接
- 建议使用国内网络

## 🐛 常见问题

### 📱 扫码登录相关问题

### Q1: 扫码登录按钮点击后无反应？

**可能原因**:
1. 浏览器未正确安装
2. Playwright 未正确配置
3. 系统资源不足

**解决方法**:
```bash
# 重新安装 Playwright 浏览器
playwright install chromium

# 检查浏览器路径
E:\chrome-win\chrome.exe
```

### Q2: 二维码不显示？

**可能原因**:
- 网络连接问题
- 平台服务器问题
- 页面加载失败

**解决方法**:
1. 检查网络连接是否正常
2. 尝试手动访问平台网站
3. 关闭浏览器窗口，重新点击扫码登录
4. 如仍不行，使用手动配置 Cookie 方式

### Q3: 扫码后手机无反应？

**可能原因**:
- 手机网络问题
- 二维码已过期
- APP 未正确识别

**解决方法**:
1. 检查手机网络连接
2. 刷新二维码重新扫描（二维码有效期约 60 秒）
3. 确保手机 APP 已更新到最新版本
4. 尝试重启手机 APP

### Q4: 扫码后提示登录超时？

**可能原因**:
- 未在手机上确认登录
- 网络延迟
- 平台服务器响应慢

**解决方法**:
1. 扫码后及时在手机上点击确认
2. 等待最多 120 秒
3. 超时后重新扫码登录
4. 检查手机和电脑的网络连接

### Q5: 提示登录成功但发布失败？

**可能原因**:
- Cookie 未正确保存
- Cookie 部分缺失
- 平台额外验证

**解决方法**:
1. 重新扫码登录
2. 检查控制台错误信息
3. 尝试手动配置 Cookie
4. 联系技术支持

### 🔧 其他常见问题

### Q6: 应用无法启动？

**检查 Python 环境**:
```bash
python --version
pip list | findstr PyQt5
pip list | findstr playwright
```

**重新安装依赖**:
```bash
pip install -r requirements.txt --force-reinstall
```

### Q7: 发布失败？

**检查 Cookie**:
1. 进入 Cookie 管理页面
2. 检查 Cookie 是否有效
3. 重新获取并保存 Cookie

**检查网络**:
1. 确认网络连接正常
2. 尝试访问对应平台网站

**查看日志**:
- 控制台会输出详细错误信息
- 截图保存错误信息便于排查

### Q8: 浏览器无法启动？

**检查 Chromium 路径**:
```
E:\chrome-win\chrome.exe
```

**确认文件存在**:
- 打开文件资源管理器
- 导航到 E:\chrome-win\
- 确认 chrome.exe 存在

**重新下载 Chromium**:
```bash
playwright install chromium
```

### Q9: Cookie 保存后仍然发布失败？

**可能原因**:
1. Cookie 已过期 - 重新获取 Cookie
2. Cookie 格式错误 - 检查格式
3. 平台要求额外验证 - 手动登录一次

**解决方法**:
1. 删除旧 Cookie
2. 重新登录平台
3. 获取新的 Cookie
4. 保存并测试

### Q10: 发布过程中浏览器被关闭？

**可能原因**:
- 网络超时
- 平台检测到自动化操作
- 系统资源不足

**解决方法**:
1. 检查网络连接
2. 降低发布频率
3. 关闭其他占用资源的程序

### Q11: 如何清空所有数据？

**删除配置目录**:
```
~/.multi_platform_publisher/
```

Windows 路径：
```
C:\Users\<用户名>\.multi_platform_publisher\
```
