# 📱 扫码登录功能说明

> 使用手机扫码快速获取 Cookie，无需手动操作

---

## 📖 功能概述

扫码登录功能允许用户通过手机扫描电脑屏幕上的二维码，自动完成平台登录并获取 Cookie。这是获取 Cookie 最简单、最安全的方式。

### 核心优势

| 特性 | 说明 |
|------|------|
| 🚀 **快速** | 30 秒内完成登录 |
| 🔒 **安全** | Cookie 自动加密存储 |
| 📱 **便捷** | 只需手机扫码 |
| 🎯 **智能** | 自动检测登录状态 |
| 💾 **自动** | Cookie 自动保存 |

---

## 🔬 工作原理

### 技术流程

```
┌─────────────────────────────────────────────────────────────┐
│                      扫码登录流程                           │
└─────────────────────────────────────────────────────────────┘

  用户点击          启动浏览器          打开登录页面
「扫码登录」   ──→   (Playwright)  ──→   (各平台)
                                              │
                                              ▼
  自动保存 Cookie   ←──  获取 Cookie   ←──  扫描二维码
  (加密存储)           (自动提取)          (手机 APP)
                                              │
                                              ▼
  关闭浏览器   ←──  登录成功检测   ←──  手机确认登录
                        (自动)
```

### 技术细节

1. **浏览器自动化**
   - 使用 Playwright 启动 Chromium 浏览器
   - 以可见模式运行（非无头模式）
   - 用户可以看到完整的登录过程

2. **二维码显示**
   - 自动导航到平台登录页面
   - 页面自动显示登录二维码
   - 无需额外操作

3. **登录状态检测**
   - 知乎：检测用户头像元素
   - 小红书：检测用户信息元素
   - 快手：检测 kpn Cookie
   - 抖音：检测 passport_csrf_token Cookie

4. **Cookie 提取**
   - 登录成功后自动提取所有 Cookie
   - 转换为标准字符串格式
   - 返回给主程序保存

5. **安全存储**
   - Cookie 使用 Fernet 加密
   - 存储到 SQLite 数据库
   - 密钥单独保存

## 🔒 安全说明

### Cookie 安全

- ✅ **加密存储**：使用 Fernet 对称加密
- ✅ **本地存储**：Cookie 仅保存在本地
- ✅ **自动过期**：Cookie 过期后自动提示重新登录
- ❌ **不上传**：Cookie 不会上传到任何服务器
- ❌ **不分享**：不要将 Cookie 分享给他人

### 存储位置

| 文件 | 路径 | 说明 |
|------|------|------|
| Cookie 数据库 | `~/.multi_platform_publisher/cookies.db` | 加密后的 Cookie |
| 加密密钥 | `~/.multi_platform_publisher/.key` | 解密密钥 |

Windows 完整路径示例：
```
C:\Users\<用户名>\.multi_platform_publisher\cookies.db
C:\Users\<用户名>\.multi_platform_publisher\.key
```

## ⚠️ 注意事项

### 1. 网络要求
- 需要稳定的网络连接
- 确保能正常访问各平台网站
- 某些地区可能需要特殊网络环境

### 2. 二维码有效期
- 二维码通常在 60 秒后过期
- 过期后页面会自动刷新或提示
- 超时请重新点击「扫码登录」

### 3. 手机 APP 要求
- 需要安装对应平台的手机 APP
- APP 需要已登录账号
- APP 需要支持扫码功能

### 4. 浏览器要求
- 需要 Chromium 浏览器
- 路径：`E:\chrome-win\chrome.exe`
- 或使用 Playwright 自动下载

### 5. 登录频率
- 不建议频繁扫码登录
- 可能触发平台安全机制
- 建议 Cookie 过期后再重新登录

## 🐛 常见问题

### Q1: 二维码不显示？

**可能原因**:
- 网络连接问题
- 平台服务器问题
- 浏览器启动失败

**解决方法**:
1. 检查网络连接
2. 尝试手动访问平台网站
3. 确认 Chromium 浏览器路径正确
4. 重新点击「扫码登录」

### Q2: 扫码后无反应？

**可能原因**:
- 手机网络问题
- 二维码已过期
- 平台检测异常

**解决方法**:
1. 检查手机网络连接
2. 刷新二维码重新扫描
3. 确保手机 APP 已登录
4. 尝试重启应用

### Q3: 登录超时？

**可能原因**:
- 扫码后未在手机上确认
- 网络延迟
- 平台服务器响应慢

**解决方法**:
1. 扫码后及时在手机上确认
2. 等待最多 120 秒
3. 超时后重新登录

### Q4: 提示登录成功但发布失败？

**可能原因**:
- Cookie 未正确保存
- Cookie 格式问题
- 平台额外验证

**解决方法**:
1. 检查 Cookie 是否已保存
2. 重新扫码登录
3. 查看控制台错误信息
4. 联系技术支持

### Q5: 如何切换账号？

**解决方法**:
1. 进入 Cookie 管理页面
2. 点击对应平台的「扫码登录」
3. 使用新账号扫码即可
4. 旧 Cookie 会被自动覆盖

### Q6: 浏览器窗口闪退？

**可能原因**:
- 系统资源不足
- 浏览器冲突
- 杀毒软件拦截

**解决方法**:
1. 关闭其他占用资源的程序
2. 暂时关闭杀毒软件
3. 重新运行应用
4. 检查系统日志

## 📊 技术规格

### 依赖库

```python
playwright >= 1.40.0    # 浏览器自动化
cryptography >= 41.0.0  # Cookie 加密
sqlite3                 # 数据存储
```

### 代码结构

```
publisher/
└── qrcode_login.py     # 扫码登录模块
    ├── QRCodeLogin     # 主类
    ├── start_browser() # 启动浏览器
    ├── close_browser() # 关闭浏览器
    ├── login_zhihu()   # 知乎登录
    ├── login_xiaohongshu()  # 小红书登录
    ├── login_kuaishou()     # 快手登录
    ├── login_douyin()       # 抖音登录
    └── login()         # 统一登录接口
```

### API 接口

```python
# 使用示例
from publisher.qrcode_login import QRCodeLogin

login_manager = QRCodeLogin()
success, cookie = login_manager.login('zhihu')

if success:
    print(f'登录成功！Cookie: {cookie}')
else:
    print(f'登录失败：{cookie}')
```
